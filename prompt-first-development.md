# Prompt-First Development

*En övning i att dokumentera processen*

---

## Bakgrund

Generativa AI-verktyg förändrar hur kod skrivs. Men verktygens effektivitet beror helt på hur väl de instrueras. Förmågan att formulera, förfina och dokumentera prompts är en professionell kompetens — lika viktig som koden som produceras.

**Denna övning fokuserar på processen, inte slutprodukten.**

---

## Uppdraget

Varje grupp väljer ett mindre utvecklingsprojekt och genomför det med hjälp av valfria AI-verktyg. Projektets omfattning är sekundär. Det centrala är att dokumentera varje steg i promptarbetet:

- Vad var intentionen?
- Hur formulerades prompten?
- Vad blev resultatet?
- Vilka justeringar gjordes, och varför?

---

## Krav på dokumentation

Följande ska finnas med i gruppens material:

**1. Projektbeskrivning**  
En kort text som definierar vad som ska byggas, vilka begränsningar som finns, och vad ett lyckat resultat innebär. Skrivs *innan* första prompten körs.

**2. Promptlogg**  
Varje prompt som används, i kronologisk ordning, med anteckningar om:
- Syftet med prompten
- Förväntad output
- Faktisk output
- Reflektion: vad fungerade, vad saknades?

**3. Processvisualisering**  
En visuell representation av arbetsflödet. Forma fritt: flödesschema, tidslinje, eller annan modell som tydliggör resan från idé till resultat.

---

## Tidsram

En arbetsdag. Förslag på uppdelning:

| Fas | Check |
|-----|-----|
| Gruppformering och val av projekt |  |
| Projektbeskrivning och förberedelse |  |
| Genomförande med löpande dokumentation |  |
| Förberedelse av presentation |  |
| Presentationer |  |

---

## Presentation

Varje grupp presenterar (5–8 minuter):

1. Projektets mål — en mening
2. Första prompten — visa den, förklara tanken bakom
3. Vad som hände — överraskningar, misslyckanden, vändpunkter
4. Den prompt ni önskar att ni börjat med — med facit i hand
5. Demonstration — i det skick det befinner sig

---

## Bedömningsgrunder

*Det som värderas är inte hur polerad slutprodukten är.*

Det som värderas:

- Tydlighet i projektbeskrivningen
- Kvalitet på promptdokumentation
- Ärlig reflektion över iterationer
- Förmåga att visualisera och kommunicera processen

---

## Att tänka på

Kod kan alltid skrivas om. En väldokumenterad process kan återanvändas, läras ut och förbättras. Det är där det långsiktiga värdet ligger.
